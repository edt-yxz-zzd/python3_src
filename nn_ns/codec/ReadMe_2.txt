其他:
  当在文件内保存一不特定长的字节串时，需要记录『长度』，为避免『长度』的『长度』，可使用『可识别结尾的整数编码』
    #以下，『*』类似 正则表达式，数字 代表 每字节的损耗比特数
    # 以下 皆能 识别结尾
    1*1         # 不能 识别开头，其余皆能
    1|32*3
    2|31*3
    21*2
    # 编码一连串整数时，并使用『中间随机落点识别所在整数』技术时，不论能否识别开头，全部需要『有效范围』，即：整数流编码后的存储空间的起止地址
    # 只能 识别结尾，相当于逗号，不能嵌套
    # 识别开头+识别结尾，相当于括号，可嵌套，故而整个『整数流』可用一括号包裹，无需『长度』或『有效范围』
    命名:
      dyb = dynamic bytes
      U = union
      s = star
      r = repeat
      t = ternimal
      1*1 => ?_1s1__1_0
        => dyb_1r0t
      1|32*3 => ?_1U32s3__0_111_10_110
        => dyb_0tU111t10r110t
      2|31*3 => ?_2U31s3__10_111_0_110
        => dyb_10tU111t0r110t
      21*2 => ?_21s2__11_0_10
        => dyb_11t0r10t
